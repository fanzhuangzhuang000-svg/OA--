# v0.3.9 P1 业务逻辑 UI 设计方案

> 文档版本：v1.0  
> 适用版本：v0.3.9 P1（销售前链路业务逻辑增强）  
> 范围：API 真实 CRUD、转化闭环、报价单 v2、跟进附件、居间费自动结算  
> 设计基线：v0.3.9 P0 已实现的 7 个页面（Leads / LeadsBoard / Opps / OppsBoard / Quotes / Referrers / ProjectPool）

---

## 一、设计原则与风格延续

### 1.1 五大设计原则

| # | 原则 | 说明 |
|---|------|------|
| 1 | **延续 P0 视觉 DNA** | 严格复用 `Leads.vue` / `Opps.vue` / `Pool.vue` 的 `page-container` + `page-header` + `filter-bar` + `content-card` 四层骨架；不在 P1 引入新容器/新色彩。 |
| 2 | **Element Plus 优先** | 任何交互优先用 `el-*` 组件（Dialog / Drawer / Form / Table / Steps / Popconfirm / Upload）；自造组件只发生在 Element 没有的「产品库选择器」「附件预览」等场景。 |
| 3 | **业务主色 #0C447C + 状态语义色** | 标题/链接/主按钮用 `#0C447C`；成功 `#1D9E75`、警告 `#BA7517`、危险 `#A32D2D`、信息 `#534AB7`；与 P0 严格一致。 |
| 4 | **单一职责页面** | 一个 URL 一个核心任务：列表只做查询/筛选/跳转，详情只做查看/编辑，弹窗只做单步操作（如转商机 / 提交审批 / 客户接受）。 |
| 5 | **可追踪的操作链路** | 所有跨实体写操作（转商机 / 成交 / 转项目 / 提交审批）必须经过 `ElMessageBox.confirm`（关键决策）或 `el-popconfirm`（轻量确认），且 UI 反馈用 `ElMessage.success` 即时给结果。 |

### 1.2 P0 风格指纹（设计语言清单）

- **页头**：`background: #fff`、圆角 8px、`box-shadow: 0 1px 4px rgba(0,0,0,0.04)`、`page-title` 字号 18px、左侧 4px 蓝色竖条。
- **统计卡**：`stats-row` 用 `grid-template-columns: repeat(4, 1fr)`（漏斗改 6 列），卡片左侧 4px 竖条，hover 抬升 2px 阴影。
- **表格**：统一 `:header-cell-style="{ background: '#f5f7fa', color: '#303133', fontWeight: 600 }"`；状态列用 `el-tag effect="dark" size="small"`，数据列用 `show-overflow-tooltip`。
- **看板**：列宽 220px，列头用半透明主题色 `rgba(color, 0.1)`，drop target 加 2px 主色边框。
- **数字/金额**：统一用 `toLocaleString('zh-CN', { maximumFractionDigits: 2 })` 格式化，金额前缀 `¥ `，主色高亮。
- **空状态**：`el-empty :image-size="50"`（小）/ `:image-size="80"`（大）。
- **错误反馈**：`ElMessage.error('加载XX失败')`，loading 状态 `v-loading` 统一管。

#### 2.1.2 三态 UI 规范（统一）

```
┌─ 加载态 ─────────────────────────────────────────────┐
│ el-table v-loading="loading"  → 表格内置骨架 8 行    │
│ el-button :loading="submitting" → 主按钮转圈        │
└─────────────────────────────────────────────────────┘
┌─ 空态 ────────────────────────────────────────────────┐
│ el-table 内置 <template #empty>                      │
│   <el-empty :image-size="80" description="暂无数据"/>│
│ 或 el-empty :image-size="50" （看板列内）            │
└─────────────────────────────────────────────────────┘
┌─ 错误态 ──────────────────────────────────────────────┐
│ ① 顶部 el-alert type="error" :closable="false"       │
│    title="加载失败：XXX"  + [重试] 按钮              │
│ ② 列表保持空（不破布局）                             │
│ ③ console 输出详细错误便于排查                      │
└─────────────────────────────────────────────────────┘
```

#### 2.1.3 新增/新建类操作的 Dialog 统一规格

- **宽度**：单列表单 520px；含表格 880px；多步骤 720px。
- **标题**：左侧主色小图标（如 `Plus` / `Edit`）+ 标题文字 + 右上角关闭。
- **底部**：左侧次要按钮（取消） + 右侧主按钮（确认/保存），主按钮 `:loading="submitting"`。
- **校验**：表单 `el-form` `:rules` 全字段必填/格式校验；提交前 `formRef.value.validate()`。

#### 2.1.4 列表/详情/表单的标准化布局

| 类型 | 布局结构 | 复用组件 |
|------|----------|----------|
| **列表页** | `page-header`（标题 + 操作） → `stats-row`（4-6 统计） → `filter-bar`（el-form inline） → `content-card`（el-table + el-pagination） | P0 全部列表页 |
| **详情页** | `page-header`（返回 + 标题 + 状态 tag + 操作） → `overview-card`（4/6 关键指标） → `el-tabs`（基本信息 / 跟进 / 附件 / 历史） → 内部 `content-card` 包裹具体内容 | 参照 `ProjectDetail.vue` |
| **新建/编辑** | el-dialog → el-form（label-width="100px" / "120px"） → 底部固定按钮 | 新组件（见 2.3 报价单） |

#### 2.2.2 转化按钮的设计规范

| 触发位置 | 按钮 | 确认方式 | 反馈 |
|----------|------|----------|------|
| 线索列表操作列 | **转商机**（success） | `el-popconfirm` 二次确认（轻量，停留在列表） | success toast + 自动 reload 列表 |
| 线索看板拖拽到「已转商机」列 | （拖拽即触发） | `ElMessageBox.confirm` 弹窗（重决策） | toast + reload 看板 |
| 商机列表操作列 | **报价**（primary） | 无确认，直接 `router.push('/sales/opps/{id}/quote')` | 路由切换 |
| 商机列表操作列 | **成交**（warning） | `ElMessageBox.confirm` 多行（重决策） | success toast + 跳转项目池 |
| 商机看板拖拽到「成交」列 | （拖拽即触发） | `ElMessageBox.confirm`（含影响说明：自动建项目池） | success toast + reload |
| 项目池列表操作列 | **转为施工项目**（success） | `ElMessageBox.confirm`（已有，沿用 P0） | success toast + reload |
| 报价单页头 | **客户接受**（primary） | `ElMessageBox.confirm`（含推进商机阶段说明） | success toast + 商机状态联动 |

**按钮类型映射**（与 P0 保持一致）：
- primary（主推）= `#0C447C` → 用于「客户接受」「提交审批」
- success（完成/转正）= `#1D9E75` → 用于「转商机」「转为施工项目」
- warning（决策/警示）= `#BA7517` → 用于「成交」
- danger（放弃/删除）= `#A32D2D` → 用于「战败」「丢弃」「删除」

#### 2.2.5 转化失败/冲突的错误提示 UI

| 失败场景 | UI 反馈 | 组件 |
|----------|---------|------|
| 线索已被其他销售抢占 | `ElMessageBox.alert`："该线索已被 [李四] 在 5 分钟前转商机，无法操作" | type=warning |
| 商机没有「已接受」报价单就推进到「成交」 | `ElMessageBox.alert`：「请先创建并提交客户接受的报价单」 | type=warning |
| 项目池已存在同名项目 | `ElMessage.error('项目「XX」已存在，请检查项目池')` + 红框高亮 | toast + el-form-item error |
| API 5xx | `ElMessageBox.alert`：「系统繁忙，请稍后重试」+ 错误码 | type=error |
| 网络断开 | `el-alert type="error" :closable="false"` 顶部条幅 + [重试] | 持久 alert |

#### 2.2.6 推荐使用的 Element Plus 组件

- `el-popconfirm`（轻量确认）
- `ElMessageBox.confirm / alert`（重决策 + 说明文字）
- `el-timeline` + `el-timeline-item`（流转历史）
- `el-progress`（漏斗条）
- `el-empty`（空漏斗）
- `el-alert`（全局错误条幅）

**技术实现**：
- 组件：自造一个 `ProductPickerDialog.vue`（基于 `el-dialog` + 内部 `el-table` 或 grid 卡片）
- 数据源：`GET /api/inventory/items?with_stock=true`（库存模块 P0 已有）
- 多选：el-table `@selection-change` 收集 id 列表，footer 显示「已选 N 项」+ 确定按钮

- 数量 / 单价 / 折扣率 列使用 `el-input-number size="small"` 直接编辑
- 实时计算「小计 = 数量 × 单价 × (1 - 折扣率)」
- 使用 `@cell-dblclick` 或始终可编辑（推荐始终可编辑，更直接）

## 五、响应式与可访问性

### 5.1 断点策略

| 断点 | 宽度 | 处理 |
|------|------|------|
| **最小支持** | 1280px（强制） | 提示用户升级屏幕（`< 1280` 显示 alert） |
| 标准桌面 | 1280–1919px | 4-6 列网格，所有功能完整 |
| 大屏 | ≥ 1920px | 漏斗/统计卡自动 spread，弹窗宽度放大到 960px |

### 5.2 大屏特殊处理

- **统计卡**：从 4 列变 6 列（已有逻辑：`grid-template-columns: repeat(auto-fit, minmax(220px, 1fr))`）
- **详情页 overview**：el-col 间距 0，列宽 `span="6"` 不变，但 padding 加倍
- **弹窗**：宽度 +120px（如 520→640，880→1000）
- **看板列宽**：220px → 280px
- **表格**：`max-height: calc(100vh - 320px)`，内容居中显示

### 5.3 颜色对比度（WCAG AA）

- 所有正文文字（#303133 / #606266 / #909399）对比白底 ≥ 4.5:1 ✅
- 主色 #0C447C 对比白底 = 10.5:1 ✅ AAA
- 链接 hover 加 underline 增强辨识
- 状态 Tag 文字与底色对比 ≥ 4.5:1（已用 Element Plus 默认值，验证通过）
- **唯一例外**：warning 文字 `#BA7517` 在白底上是 3.9:1，低于 AA → 改用 `#A06D14` 或加 `text-shadow` 提升
- 危险状态 `#A32D2D` = 6.8:1 ✅
- 成功 `#1D9E75` = 3.5:1（接近 AA）→ 改用更深的 `#0F6E56` 提至 5.2:1

**无障碍补强**：
- 所有 `el-button` 有可见的 `aria-label`（如 icon-only 按钮）
- `el-dialog` 默认 focus 第一个输入框
- `el-form` 校验失败的字段 `aria-invalid="true"`
- 表格行支持键盘导航（Element Plus 默认已支持）

### 5.4 加载/无障碍交互

- 所有 loading 状态显示 `el-skeleton` 或 `el-icon class="is-loading"`（旋转）
- 异步操作有 timeout 兜底（10s 后提示重试）
- 所有破坏性操作（删除/驳回/丢弃）有 undo 撤销按钮（toast 内 [撤销]）

## 六、交互流程图

### 6.1 API 真实 CRUD 流程

```
用户点击 [转商机]
    ↓
el-popconfirm 显示「确认将「前海科技园」转为商机？」
    ↓ (确认)

    │           ↓
    │         reload() 列表 + 自动跳转 /sales/opps/{newId}
    │
    └─ 4xx/5xx → ElMessageBox.alert(错误信息, '操作失败', { type: 'error' })

              列表保持原状，无副作用
```

### 6.2 转化闭环流程（线索→商机→项目池→项目）

```
[线索] 拖拽到「已转商机」列

ElMessageBox.confirm 弹窗「将创建商机，是否继续？」

            商机自动进入「需求确认」阶段

              ├─ 取消 → 不动
              └─ 确认 → POST /api/sales/opps/{id}/win
                        ↓
                      项目池新增一条记录
                        ↓
                      ElMessage.success('已建项目池，等待分配项目经理')
                        ↓
            [项目池] 点击「转为施工项目」
              ↓
            ElMessageBox.confirm「确认转为施工项目？将自动建好项目档案」
              ├─ 取消 → 不动

                      /projects 表新增记录 + 跳转 /project/detail/{id}
                        ↓
                      ✅ 转化闭环完成
```

### 6.3 报价单 v2 创建/编辑流程

```
[商机详情] 点击 [报价]
    ↓
跳 /sales/opps/{id}/quote
    ↓
点击 [+ 新建版本]（沿用 P0）或 [V1 草稿] 进入编辑
    ↓
[基本信息] Tab：填写客户/商机/有效期（自动带出）→ 切到 [产品清单]
    ↓
[产品清单] Tab：
    点击 [从产品库选择] → 弹 ProductPickerDialog
        ├─ 多选产品 + 设置数量

    行内编辑 数量 / 单价 / 删除
    ↓
[价格/税额] Tab：
    实时预览自动计算（响应式）
    选择折扣模式（率/额）+ 税率
    确认含税总价
    ↓
点击 [保存草稿] → POST /api/sales/quotes 草稿状态
    ↓
点击 [提交审批] → ElMessageBox.confirm
    ↓
POST /api/sales/quotes/{id}/submit
    ↓
状态：draft → submitted（Tag 颜色变化）
    ↓
[客户接受] 按钮：

    ├─ 200 → 商机 stage → contracting

```
