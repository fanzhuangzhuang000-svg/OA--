# 172.20.0.139 服务器连接指南

## 🔍 问题诊断

SSH 密钥认证失败，可能原因：
1. 服务器上的公钥被删除
2. 私钥文件权限不正确（Windows 上 chmod 无效）
3. 服务器 SSH 配置变更

---

## ✅ 解决方案

### 方案 1：手动连接并重新配置 SSH 密钥

**步骤 1：手动连接服务器**

打开 **Git Bash** 或 **PowerShell**，运行：

```bash
ssh nbcy@172.20.0.139
```

输入密码（如果你知道密码）。如果不知道密码，尝试以下常见密码：
- `admin123`
- `Aa782997781.`
- 或其他你可能设置过的密码

**步骤 2：重新添加公钥**

如果连接成功，在服务器上运行：

```bash
# 创建 .ssh 目录（如果不存在）
mkdir -p ~/.ssh
chmod 700 ~/.ssh

# 添加公钥（将本地的 ~/.ssh/id_rsa.pub 内容添加到服务器的 ~/.ssh/authorized_keys）
echo "YOUR_PUBLIC_KEY_HERE" >> ~/.ssh/authorized_keys

# 设置权限
chmod 600 ~/.ssh/authorized_keys
```

**步骤 3：获取本地公钥**

在本地电脑上运行：

```bash
cat ~/.ssh/id_rsa.pub
```

复制输出的内容，粘贴到步骤 2 的 `YOUR_PUBLIC_KEY_HERE` 位置。

---

### 方案 2：启用密码认证（临时）

如果你能连接到服务器，可以临时启用密码认证：

**步骤 1：修改 SSH 配置**

```bash
sudo nano /etc/ssh/sshd_config
```

找到以下行，修改为：

```
PasswordAuthentication yes
```

**步骤 2：重启 SSH 服务**

```bash
sudo systemctl restart sshd
```

**步骤 3：测试密码连接**

```bash
sshpass -p 'your_password' ssh nbcy@172.20.0.139 "whoami"
```
