## 🐛 发现的问题

### 1. PostgreSQL 序列不同步
- **现象**: 创建应收款/施工日志时 500 错误（duplicate key）
- **根因**: 序列值落后实际最大 ID
- **修复**: 运行 `fix_seq.sql` 手动修复序列
- **状态**: ✅ 已修复

### 2. 应收款关闭逻辑
- **现象**: 收款到位后，手动关闭返回 422
- **分析**: 系统已自动处理，无需手动关闭（符合业务规则）
- **建议**: 前端在收款到位后自动刷新状态，不显示"关闭"按钮

### 3. 应收款列表分页
- **现象**: 新创建的应收款可能不在第一页
- **建议**: 创建成功后跳转到详情页，或在列表中高亮新创建的项
