## 📚 经验教训总结

### 1. 问题排查流程（重要！）

❌ **错误做法**：
- 直接修改代码并部署
- 没有先确认问题根源
- 多次修改后才测试

✅ **正确做法**：
1. **复现问题**：浏览器 F12 查看 Console 和 Network
2. **检查后端日志**：`tail -f /var/www/oa-api/storage/logs/laravel.log`
3. **隔离测试**：用硬编码数据测试（确认前后端连接是否正常）
4. **逐步修复**：先让 API 返回简单数据，再逐步增加复杂度
5. **验证修复**：清除浏览器缓存，重新测试

---

### 2. Laravel Enum 处理（踩坑！）

❌ **踩坑代码**：
```php
// 错误：类型声明为 ?string，但 $project->stage 返回的是 Enum 对象
private function formatStage(?string $stage): string
{
    $map = ['initiation' => '立项', ...];
    return $map[$stage] ?? $stage ?? '未知';
}
```

✅ **正确做法**：
```php
// 正确：接受混合类型，并处理 Enum
private function formatStage($stage): string
{
    // 处理 Enum 对象
    if ($stage instanceof \App\Enums\ProjectStage) {
        $stage = $stage->value;
    }
    
    $map = [
        'initiation'   => '立项',
        'inquiry'      => '询价',
        // ...
    ];
    
    return $map[$stage] ?? $stage ?? '未知';
}
```

**经验教训**：
- Laravel 10+ 的 Eloquent 模型如果使用 Enum 类型转换，`$model->field` 返回的是 **Enum 对象**，不是字符串
- 需要检查模型是否有 `protected $casts = ['stage' => ProjectStage::class]`
- 或者在控制器中用 DB 查询（返回字符串），避免 Enum 类型转换

### 3. 前后端数据格式不匹配

❌ **问题根源**：
- 后端 `projectProgress()` 返回的是**项目阶段统计**（用于图表）
- 前端期望的是**项目列表**（用于表格）
- 导致前端表格没有数据

✅ **正确做法**：
- 后端和前端约定好 API 返回格式
- 用 Postman 或 curl 测试 API，确认返回数据格式
- 前端用模拟数据开发，避免依赖后端

**经验教训**：
- 修改后端 API 返回格式时，必须同步修改前端代码
- 或者保持向后兼容（不修改原有 API，新增一个 API）

### 4. 部署流程（标准操作）

✅ **标准部署流程**：
1. 修改代码
2. 上传到服务器（`sftp.put()`）
3. 复制到目标位置（`sudo cp`）
4. 修改权限（`sudo chown www-data:www-data`）
5. **重启 PHP-FPM**（关键！`sudo systemctl restart php8.3-fpm`）
6. 清除日志（`sudo truncate -s 0 /var/www/oa-api/storage/logs/laravel.log`）
7. 测试

❌ **常见错误**：
- 忘记重启 PHP-FPM（opcache 缓存旧代码）
- 忘记修改文件权限（www-data 无法读取）
- 没有清除日志（看不到最新错误）

---

### 5. 浏览器缓存问题

❌ **问题**：部署后前端没有变化

✅ **解决方法**：
- 强制刷新：`Ctrl + F5`（Windows）或 `Cmd + Shift + R`（Mac）
- 清除浏览器缓存：F12 → Application → Clear storage → Clear site data
- 后端添加 Cache-Control 头：
```php
return response()->json(...)
    ->header('Cache-Control', 'no-cache, no-store, must-revalidate')
    ->header('Pragma', 'no-cache')
    ->header('Expires', '0');
```

### 6. 调试技巧（高效！）

✅ **高效调试方法**：
1. **后端日志**：`tail -f /var/www/oa-api/storage/logs/laravel.log`
2. **前端控制台**：F12 → Console（查看 JavaScript 错误）
3. **前端网络**：F12 → Network（查看 API 调用状态）
4. **隔离测试**：用硬编码数据测试，确认前后端连接正常
5. **逐步调试**：先让 API 返回简单数据，再逐步增加复杂度

❌ **低效调试方法**：
- 反复修改代码并部署（盲目试错）
- 不查看日志（不知道错误原因）
- 不隔离问题（前后端一起改，不知道哪边有问题）

### 7. 代码质量改进建议

✅ **改进建议**：
1. **类型安全**：方法参数不要用 `?string`，用 `mixed` 或具体类型
2. **错误处理**：添加 try-catch，返回友好错误信息
3. **日志记录**：关键步骤添加日志（`Log::info()` 或 `Log::error()`）
4. **单元测试**：为关键方法写测试（避免回归）
