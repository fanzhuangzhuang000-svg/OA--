## 06-20 17:00 OA 主页打不开 — 部署脚本 Windows 路径 bug

### 症状
- http://172.20.0.139 主页 200 但白屏
- index.html 200 size 445 ✅
- /assets/index-*.js 200 size 445 ❌ (实际拿的是 index.html)
- 浏览器 F12 看 Network: assets 全部 404

### 根因 (重要避雷)
**`/var/www/oa-web/` 顶层有 197 个 `assets\xxx.js` 文件 (反斜杠当文件名字符了)**, 但 `assets/` 目录不存在
- 历史部署用 `os.path.join(local, name)` + sftp.put
- Windows Python 跑 `os.path.join` 用 `\` 拼接
- sftp.put 收到 `D:\...\dist\assets\index-xxx.js` 这种路径, 把整个字符串当 localpath
- 但 Windows 路径分隔符在 Linux 远端, 文件落到 `/var/www/oa-web/assets\index-xxx.js` (反斜杠在文件名里)
- nginx try_files 找不到 `/var/www/oa-web/assets/index-xxx.js` (因为这是文件名不是目录), SPA fallback 回 index.html (445B)

### 紧急修复
```bash
cd /var/www/oa-web
mkdir -p assets
for f in 'assets\'*; do
  [ -f "$f" ] && mv "$f" "assets/${f#assets\}"
done
```
- 197 个文件 30 秒全 mv 到真 assets/ 目录
- nginx 立即能找到

### 一劳永逸修复
- `.workbuddy/deploy_web.py` 重写: 用 `Path.as_posix()` + `posixpath.join` 替代 `os.path.join`
- `.workbuddy/deploy_web_172.py` 重写: 同样修法
- `.workbuddy/deploy_to_172.py` 修: `os.path.join(dirpath, fn)` → `posixpath.join(dirpath.replace('\\', '/'), fn)`

### 教训
**Windows 上 sftp.put 任何路径都必须先 `as_posix()` 或 `replace('\\', '/')`**, 永远不要相信 `os.path.join` 在跨平台场景

- 修复方案: 必须显式 `$table` (如果表名不规则)
- 两台服务器 (172 + 152) 都同步了这个 fix

**经验教训**:
