## 20:53 修复员工管理新建员工 500 错误

**根因**: 3 个 bug 叠加
1. `bcrypt()` + User 模型 `'password' => 'hashed'` cast → 密码双重哈希 (不会直接抛500,但密码不可用)
2. `phone` 列 NOT NULL UNIQUE，不填时被设为 `''`，第二个空 phone 用户触发 PG 23505 唯一约束冲突
3. 修复后生成占位 phone `TEL-{username}-{6hex}` 超过 VARCHAR(20) → PG 22001 截断错误

**最终修复** (`EmployeeController@store` line 76-80):
- 移除手动 `bcrypt()`，让模型 `hashed` cast 自动加密
- 不填 phone 时生成 `mb_substr('TEL-{username}-{4hex}', 0, 20)` 确保 ≤20 字符
- 同步修复 `import()` 方法相同的问题

**验证**: 152 服务器 `POST /api/employees` 不填 phone → code:0 创建成功

## 教训 (给后续 172/双服务器)
- **永远先查 schema, 再生成数据**: `\\d table_name` 看实际列名, 不要假设 152/172 字段一致
- **172 的 Laravel enum 比 152 严**: 7 个 stage 值 (contract/initiation/inquiry/construction/settlement/purchase/warranty), 写错必 500
- **172 必走 `sudo -n -u postgres`**: oa_user 没 superuser, 没法用 `SET session_replication_role`
- **每张表一个独立 SQL block**: 单表失败不阻塞后续, 这种"宽容"对回填特别重要
- **`finance_accounts` 在 172 没 unique constraint**: 不能 `ON CONFLICT (account_no)`, 改 `WHERE NOT EXISTS`
- **certificates 在 172 无 unique constraint (除 id)**: 同样 `WHERE NOT EXISTS`
- **`employee_profile.id=2` 在 172 缺**: 用 `[1,3,4,5,6,7,8,1]` 数组取
- **152 的 ENUM 引用 `App\\Enums\\ProjectStage` 但文件不存在**: 类找不到时 Laravel 似乎做兜底, 152 跑 200, 172 也跑 200 (我猜 Laravel 用 try/catch 包了 from())
- **python 字符串里的 `%`**: f-string 不能直接放 `%{...}` curl format, 用单引号或 f-string 加反斜杠
- **172 路由命名风格和 152 不同**: 152 是 `api/expense-claims`, 172 是 `api/expenses`; 152 是 `api/leads`, 172 是 `api/sales/leads`. 同步测试要查 `route:list` 而不是猜
