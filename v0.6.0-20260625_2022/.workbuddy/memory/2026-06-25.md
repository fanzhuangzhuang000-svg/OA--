## V0.5.8.4 维修工单 res.data 误读 (16:20)
**现象**: "Cannot read properties of undefined (reading 'code')" 新建维修工单报错
**根因**: `request.ts` 84-86 行已把 `res.data` 解包, 但前端仍按未解包用 `res.data.code`
**修法** (3 个文件):
- `RepairCreate.vue::onSubmit` - `res?.data ?? res` 双兼容
- `WorkOrderCreate.vue` - 同
- `WorkOrderDetail.vue::onConvertConfirm` - `result.repair_order || result`

**如何系统检测同类 BUG** (3 个动作):
1. grep 模式: `res\.data\.(id|code|token|user|items)` - 找"按未解包用"的可疑点
2. puppeteer 模拟真实提交: 登录 → 进表单 → type → 点提交 → 听 pageerror/response
3. 拦截器加防御: `res?.data ?? res` 让历史代码不报错

**关键习惯**:
- 任何 `await get/post/put/patch/del(...)` 后, 直接当 `res = 后端的 data` 用, 不要 res.data
- 例外: `e.response.data.message` 那种 error 路径不会被解包

**关键教训 — Vue 3 props/template ref 同名是大坑**:
- `<script setup>` 里, 模板 ref 名 和 prop 名 不能重复
- 编译器不会报, 但运行时 el-form ref 会被 prop 覆盖, 行为诡异

# 2) 跨文件: 父级 :some-ref="x" 传 prop, 子组件内同名 ref
parent_calls = re.findall(r':([a-zA-Z][\w-]*-ref)\s*=\s*"(\w+)"', t)
