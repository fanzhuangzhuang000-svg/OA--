# V0.5.7 块 1 完工 — 2026-06-24 18:45

## 交付内容

### 后端
1. **GET /api/projects/{id}/maintenance** — 拿一个项目下所有售后记录 (工单 + 返修单 + 汇总)
   - 返回 `items[]` (work_order / repair_order 混合, 按时间倒序)
   - 返回 `stats` (工单数/返修数/在修数/总成本)
   - 返回 `can_create_maintenance` (项目阶段校验)
2. **POST /api/work-orders** — 加 `project_id` 阶段校验
   - 只有 `settlement` / `warranty` 阶段才能创建工单
   - 否则返 422 + 错误信息
3. **POST /api/repair-orders** — 同样加 `project_id` 阶段校验
4. **POST /api/work-orders/{id}/convert-to-repair** — 转返修时**自动写项目 timeline**
   - 写入 `construction_logs` 表 (`log_type='maintenance'`)

### 前端 — 项目详情页
- 新增"售后记录" tab
- 4 统计卡 (工单/返修/在修/总成本)
- 2 按钮 (新建工单/新建返修) — 受阶段限制 (构造/采购阶段 disabled)
- 列表 (统一格式, 区分工单/返修, 显示来源)
- 阶段警告条 (非 settlement/warranty 阶段显示)
