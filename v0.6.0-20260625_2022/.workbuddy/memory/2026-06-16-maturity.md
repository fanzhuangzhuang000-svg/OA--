## 四、关键风险点（影响"上线"）

1. **数据持久化**：所有 5 个用户 + 客户 + 项目是 seeder 数据，**没用户真实数据**
2. **权限隔离**：没有角色权限分配页面；Spatie 角色有了但 `isAdmin` 之类硬编码
3. **审计**：登录、修改、删除没有操作日志（system_logs 表结构有，调用没接）
4. **文件上传**：DiskController 写了一半（hashName 写入但没返回 file_url）
5. **大屏是 demo**：6 个 metric 全是硬编码字符串
6. **业务深度**：每个模块都"刚到能 CRUD"，**缺业务规则**（如项目阶段流转、工单 SLA、报销审批流）

## 五、建议路线图（接 B → C 后）

### 短期（1-2 周）P0 打磨
- 工作台：把 dashboard/stats 接真实数据，Echarts 图表
- 系统设置：角色权限分配页 + 系统名称后端化
- 审计：所有 model 加 observer 写 system_logs
- 性能：前端主 bundle 拆 chunk

### 中期（1-2 月）P1 业务深度
- 库存：扫码出入库
- 知识库：富文本编辑器
- 财务：凭证生成
- 大屏：接真实数据源（revenue / project / service 真实 SQL）

### 长期（季度级）多端 + 培训
- 微信小程序
- 培训管理
- PC 客户端（Electron）
