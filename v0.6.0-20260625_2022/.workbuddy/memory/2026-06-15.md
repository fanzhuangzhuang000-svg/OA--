### 修复的关键问题
1. **config/app.php Provider 兼容** - Laravel 11+ 移除了 AuthServiceProvider/RouteServiceProvider/EventServiceProvider，需手动从配置中移除
2. **bootstrap/app.php API 路由** - 需添加 `api: __DIR__.'/../routes/api.php'` 到 `withRouting()`
3. **PSR-4 违规** - 多个类在同一文件中（ModuleControllers.php），用 `composer.json` 的 `files` autoload 解决
4. **DB_CONNECTION=sqlite → mysql** - Laravel 11 默认 sqlite，需手动改 .env
5. **MySQL 用户权限** - `localhost` 和 `127.0.0.1` 在 MySQL 中是不同 socket，需分别创建用户
6. **缺少 Composer 包** - 需安装 laravel/sanctum + spatie/laravel-permission
7. **personal_access_tokens 表缺失** - Sanctum 迁移被 "IN PRODUCTION" 阻止，需 `--force`；且 Spatie 迁移因已有 permissions 表冲突，需 DROP 后重建
8. **AuthServiceProvider 未注册** - 根本原因！config/app.php 的 providers 数组缺少 `Illuminate\Auth\AuthServiceProvider::class`，导致 auth Facade 不可用
9. **缺失中间件文件** - `app/Http/Middleware/Authenticate.php` 和 `RedirectIfAuthenticated.php` 不存在，需手动创建
10. **User 模型 status 枚举** - 枚举类型导致 `$user->status !== 'active'` 永远为 true，AuthController 需兼容枚举值比较
