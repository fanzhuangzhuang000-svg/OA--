**重要坑 (e2e 流程)**:
1. **status enum**: `completed` 不是合法值, 应为 `closed` (work_orders / repair_orders 都要)
2. **PG sequence 滞后**: 跑完 SQL 后必须 `setval(pg_get_serial_sequence(t,'id'), MAX(id), true)`
3. **work_orders 缺列**: `customer_signed_at` + `customer_signature_ip` (需要 ALTER TABLE 加, 真 bug)
4. **状态机**: repair_orders 必须 received → sent_for_repair → in_repair → repaired → sent_back → closed
5. **PortalRepairController.methodLabel**: 入参是 enum, 但函数签名要 string — 加 `is_string($x) ? $x : $x->value`
6. **工单 resolve**: 上门服务必须 `customer_signature` (data:image/png;base64,...)
7. **端点路径**: 维修 `/work-orders/{id}/{assign|start|resolve}` (不是 /status)
8. **销售端点**: `/api/sales/leads` (不是 /leads), `mark-won` 需要 contract_amount + signed_at
9. **oppsStore**: presale_id 必填
10. **leadsConvertToOpp**: sales_id + presale_id 都必填
11. **markInRepair**: 不接 body, 状态机: 只能从 sent_for_repair 进
12. **ship-out / ship-back**: 8 个字段必填 (sender_name/address + receiver_name/address + carrier/tracking_no)
13. **expense approve**: `action` in [approved, rejected]
