1. **MySQL enum 错配**（C3 教训最重）
   - 现象：scree 接口 500 报 "Unknown column 'scheduled_at' in 'where clause'"
   - 根因：原 DashboardController 的 `serviceMetrics()`、`projectStatus()`、`todos()` 全部用了 **不存在或语义错的列名**
   - 真实 enum：
     - `projects.stage` = initiation/inquiry/contract/purchase/construction/settlement/warranty
     - `projects.status` = pending/in_progress/completed/cancelled
     - `service_orders.status` = pending/assigned/in_progress/completed/confirmed/archived/cancelled
     - `receivables.status` = pending/partial/fully_paid/overdue
     - `attendance_records.status` = normal/late/early_leave/absent/field_work/leave
     - `expense_claims.status` = draft/submitted/approved/rejected/paid/cancelled
     - `maintenance_contracts.status` = active/expiring/expired/terminated
     - `certificates.status` = valid/expired/revoking
   - **方法**：用 `php artisan tinker` + `DB::select('SHOW COLUMNS FROM xxx')` 一次拉全表，输出 Field/Type/Null/Default 四列即可拼出 SQL
   - 修复后 C3 E2E：metrics 6 + revenueChart 12 + projectStatus 7 + serviceMetrics 3 + todos 6 全部 200

2. **公网 SSH 22 端口突然 refused**（2026-06-16 16:xx 发生）
   - 现象：`47.108.221.150:22` Connection refused（之前一直通）
   - 原因：阿里云/ISP 临时封禁或安全组规则变更

   - 教训：内网 SSH 应当作主用通道，公网仅作应急

   - 后续所有测试脚本已纠正

## v0.3.7.1 — 401 Route [login] not defined 修复（2026-06-16 17:36）

### 用户反馈
"服务端数据是否已经更新？web服务是否重启？"

### 排查发现
- 服务都活着：nginx 3000/3001 + php-fpm master 834678
- code 已更新：AuthController.php、routes/api.php、Profile.vue、Password.vue
- 但 profile 调用 500（log: `Route [login] not defined`）
- change-pw 之前看到的 302 是 validation 422 重定向，不是真的 redirect

### 根因
Laravel 11 + Sanctum 的默认 `Authenticate` 中间件，在 `expectsJson()` 返回 false 时调 `route('login')` → 抛 `RouteNotFoundException`（项目无 web login 路由）

### 修复 (3 处)
1. `pc-api/app/Http/Middleware/Authenticate.php` 新建 — `redirectTo()` 永远返回 null
2. `pc-api/bootstrap/app.php` `$middleware->alias(['auth' => 自定义])` 覆盖默认
3. `pc-api/bootstrap/app.php` `$exceptions->render(AuthenticationException...)` → 401 JSON
4. 兜底 `$exceptions->render(RouteNotFoundException 'login'...)` → 401 JSON (无 Accept header 时也工作)

### E2E 4/4 (test_full.sh)
| 测试 | 结果 |
|---|---|
| [1] change-pw wrong (min:6) | 422 + JSON `原密码至少 6 位` |
| [2] no token + Accept | 401 + JSON `未认证,请先登录` |
| [3] no token + NO Accept | 401 + JSON (兜底) |
| [4] change-pw ok | 200 OK |

### 部署 + 重启
- 上传 2 文件 (bootstrap/app.php + Authenticate.php) → `kill -USR2 834678` (php-fpm 优雅 reload)
- 新 worker pid 838439/838440 (00:03 elapsed)
- admin 密码被测试改密后用 reset_pw.php 重置回 `admin123`

### 关键经验
- **Laravel 11 纯 API 项目** 必须 显式 覆盖 `auth` 别名 或 注册 `Authenticate` 中间件，否则 Sanctum 401 触发 `route('login')` 500
- **E2E 测试** 需带 `Accept: application/json` header（前端 Axios 默认带；curl 不带）
- **php-fpm reload**: `kill -USR2 <master_pid>` 不是 `kill -HUP`（HUP 是 nginx）
- **ENUM 字段改类型后**：必须 reload php-fpm 让 worker 重新连接 MySQL 才生效（worker pool 有连接缓存）
