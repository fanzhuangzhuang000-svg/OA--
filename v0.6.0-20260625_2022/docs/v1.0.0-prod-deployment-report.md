## 🔧 关键配置

### 1. 前端构建
```bash
cd D:/work/website/OA/pc-web
npm run build  # 10.25s
# 输出：dist/ (已生成)
```

### 2. 前端部署
```bash
# 备份
cd /var/www && sudo cp -r oa-web oa-web-bak-20260622-1900

# 上传
# 本地：oa-web-dist.tar.gz → 152:/tmp/oa-web-dist.tar.gz

# 解压
cd /var/www/oa-web && sudo tar -xzf /tmp/oa-web-dist.tar.gz --strip-components=1
sudo chown -R www-data:www-data /var/www/oa-web/
```

### 3. 后端部署
```bash
# 备份
cd /var/www && sudo cp -r oa-api oa-api-bak-20260622-1900

# 上传
# 本地：oa-api-code.tar.gz → 152:/tmp/oa-api-code.tar.gz

# 解压（覆盖代码目录，保留 .env 和 storage/）
cd /var/www/oa-api && sudo tar -xzf /tmp/oa-api-code.tar.gz --overwrite
sudo chown -R www-data:www-data /var/www/oa-api/

# 验证 .env 还在
cat /var/www/oa-api/.env | grep -E "APP_ENV|APP_DEBUG|DB_DATABASE"
# 输出：APP_ENV=production, APP_DEBUG=false, DB_DATABASE=security_oa
```

### 4. 数据库迁移
```bash
cd /var/www/oa-api
sudo -u www-data php artisan migrate --force

# 报错：approval_records_v2 表已存在
# 修复：标记 migration 为已执行
sudo -u postgres psql -d security_oa -c "
INSERT INTO migrations (migration, batch) 
SELECT '2024_01_05_000004_create_approval_records_v2_table', MAX(batch)+1 
FROM migrations 
ON CONFLICT (migration) DO NOTHING;
"
```

### 5. 重启 PHP-FPM
```bash
sudo systemctl restart php8.3-fpm
```
